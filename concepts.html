Essential Terms

Important Terms Used Throughout This Site

HTTP

The Hypertext Transfer Protocol, the core application-layer protocol used for communication between clients (browsers) and servers on the Web. It uses a request/response model.

HTTPS

Secure version of HTTP that uses TLS encryption to ensure confidentiality, integrity, and authentication. Required for modern websites and APIs.

Request / Response

The fundamental structure of HTTP communication: a client sends a request (method, URL, headers), and the server returns a response (status code, headers, body).

Status Codes

Three-digit codes that indicate the result of a request. Examples include: 200 OK, 301 Moved, 404 Not Found, and 500 Server Error.

QUIC

A modern transport protocol built on UDP. QUIC integrates TLS, reduces connection setup time, and prevents TCP-style head-of-line blocking. Used by HTTP/3.

Multiplexing

A performance feature of HTTP/2 allowing multiple independent requests and responses to share a single connection, reducing latency and improving page load speed.

REST

Representational State Transfer — a popular architectural style used for designing web APIs. REST uses standard HTTP methods like GET, POST, PUT, DELETE to operate on resources.

Headers

Metadata included in HTTP requests and responses. Headers specify important details such as content type, cookies, authentication, caching behavior, and connection rules.
