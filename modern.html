2015

HTTP/2 — Faster Loading Through Multiplexing

Standardized in 2015, HTTP/2 aimed to significantly improve performance over HTTP/1.1. It introduced a more efficient way of delivering multiple resources simultaneously, reducing latency and making modern websites faster.

  • Multiplexing: Allows multiple requests/responses to be processed over one connection.
  • Header Compression (HPACK): Reduces redundant HTTP header data.
  • Server Push: The server can send files before the browser asks for them.
  • Binary Framing: Replaces text-based messages for greater efficiency.
Although HTTP/2 uses multiplexing, it still relies on TCP, which can suffer from head-of-line blocking when packets are lost.
2022

HTTP/3 — A Modern Protocol Built on QUIC

HTTP/3 represents a major redesign of how the Web moves data. Instead of relying on TCP, HTTP/3 uses QUIC — a transport protocol built on UDP. QUIC embeds TLS 1.3 directly into the transport layer, enabling faster, more secure connections.

  • UDP-based transport: Avoids TCP’s head-of-line blocking.
  • TLS 1.3 built in: All HTTP/3 connections are encrypted by default.
  • 0-RTT handshakes: Returning users connect almost instantly.
  • Better for mobile networks: Recovers faster from packet loss.
QUIC's ability to handle lost packets independently makes HTTP/3 ideal for mobile and Wi-Fi connections.
Real-World Adoption

How Modern HTTP Affects Real Websites

HTTP/2 and HTTP/3 are widely used today by major platforms, improving loading performance for billions of users around the world.

  • YouTube: Streams video using QUIC (HTTP/3) for lower latency.
  • Cloudflare: One of the earliest supporters of HTTP/3 deployment.
  • Akamai: Uses HTTP/2/3 to accelerate global content distribution.
  • Chrome, Safari, Firefox: All mainstream browsers support HTTP/2 and HTTP/3.
Over half of today's global web traffic is delivered using HTTP/2 or HTTP/3.
Security

Connections & Security in Modern HTTP

Modern versions of HTTP assume encryption by default. Browser vendors and security standards strongly encourage the use of HTTPS over plain HTTP.

  • HTTPS everywhere: Data is encrypted and protected from tampering.
  • TLS 1.3: Provides faster, more secure handshakes.
  • HSTS: Forces browsers to always connect securely.
Without HTTPS, sensitive data such as passwords or personal information can be easily intercepted.
